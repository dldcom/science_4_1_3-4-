# 화산·현무암 학습 웹앱 개발 기록

## 1. 프로젝트 목표

학생이 화산 분출부터 현무암 생성까지의 과정을 직접 관찰할 수 있는 태블릿용 3D 학습 웹앱을 만든다.

핵심 학습 내용:

- 마그마가 지표 밖으로 나오면 용암이라고 부른다.
- 용암은 지표에서 빠르게 식어 현무암이 된다.
- 화산가스가 빠져나간 일부 자리에는 기공이 생길 수 있다.
- 같은 화산이 여러 번 분출하면 이전 현무암 위에 새로운 용암이 흐를 수 있다.

## 2. 기술 선택

- 3D 엔진: Three.js
- 화면: HTML, CSS, JavaScript
- 지형: Three.js PlaneGeometry를 변형한 절차적 화산
- 표면: CC0 PBR 텍스처
- 용암: 지형 추적 경로와 커스텀 셰이더
- 효과: Points 파티클

실시간 유체 시뮬레이션은 태블릿에서 부담이 크기 때문에 사용하지 않았다. 대신 버튼을 누르는 순간 지형의 높이를 비교해 용암 경로를 계산한다.

## 3. 화산 지형 제작

1. 평면 지오메트리를 112개 세그먼트로 나눴다.
2. 원뿔 높이, 분화구 함수, 능선, 노이즈를 조합해 화산 형태를 만들었다.
3. Poly Haven의 Gray Rocks PBR 텍스처를 반복 적용했다.
4. Color, NormalGL, Roughness 지도로 색상·표면 굴곡·거칠기를 표현했다.

이 방식은 외부 화산 모델보다 용암 경로, 분화구, 향후 지하 단면을 자유롭게 제어할 수 있다는 장점이 있다.

## 4. 용암 경로 제작

1. 분화구 둘레에서 시작 방향을 선택한다.
2. 현재 지점 주변의 여러 후보 위치 높이를 비교한다.
3. 가장 자연스럽게 낮아지는 방향을 선택한다.
4. 약간의 노이즈를 더해 완전한 직선이 되지 않도록 한다.
5. 경사가 완만한 장소에서는 폭이 넓어지도록 계산한다.

초기에는 고정된 네 갈래를 사용했다. 이후 분출할 때마다 3~5개의 방향, 폭, 길이가 달라지도록 변경했다.

## 5. 텍스처 조사와 적용

### 화산 지형

- 선택: Poly Haven Gray Rocks
- 이유: 자연 암석 표면이며 CC0이고 1K PBR 지도를 제공한다.

### 뜨거운 용암

- 선택: ambientCG Lava004
- 이유: 밝고 주황색인 용암 표면이 분화구에서 막 나온 용암에 적합하다.
- 사용 지도: Color, Emission, NormalGL, Roughness

### 식어가는 용암

- 선택: ambientCG Lava002
- 이유: 검은 껍질과 붉은 균열이 함께 보여 냉각 중간 단계에 적합하다.
- 사용 지도: Color, Emission, NormalGL, Roughness

### 현무암

- 선택: ambientCG Rock041
- 비교 후보: Rock031, Rock033, Rock035, Rock037
- 선택 이유: 다른 후보보다 큰 절벽 방향성이 적어 긴 용암 리본에 반복 적용하기 좋다.
- 사용 지도: Color, NormalGL, Roughness

변위 지도는 지오메트리 수를 늘릴 수 있어 런타임에서 사용하지 않는다. 필요한 1K 지도만 프로젝트에 포함했다.

## 6. 용암 재질 전환

현재 재질 전환 순서:

1. 용암이 흐르는 동안에는 전체가 Lava004 상태를 유지한다.
2. 모든 용암 경로가 끝까지 도달하면 냉각 타이머를 시작한다.
3. 분화구에서 먼 산 아래쪽부터 Lava002가 서서히 섞인다.
4. 냉각이 계속되면 Rock041 현무암으로 바뀐다.
5. 셰이더의 해시 패턴으로 일부 위치에만 어두운 기공을 표현한다.

초기 구현에서는 흐르는 도중부터 Lava002로 바뀌고 분화구 근처부터 굳는 문제가 있었다. 냉각 시작 시점과 UV 방향 계산을 수정해 해결했다.

## 7. 부드러운 용암 흐름

초기에는 용암 리본의 삼각형 조각을 하나씩 표시했다. 이 방식은 실제 FPS가 충분해도 용암 끝이 계단식으로 나타났다.

개선 방식:

- 전체 용암 리본 지오메트리는 항상 렌더링한다.
- 셰이더에 연속적인 진행 위치 `uReveal`을 전달한다.
- 진행 위치보다 앞쪽 픽셀만 숨긴다.
- 전면 위치에 작은 노이즈를 적용해 너무 반듯한 선이 되지 않도록 한다.

이 방식은 추가 지오메트리 없이 매 프레임 부드럽게 움직인다.

## 8. 단발 분출과 반복 분출

분출 한 번의 흐름:

1. `분출 시작하기` 클릭
2. 분화구 폭발 플래시와 입자 표시
3. 새로운 용암 경로 3~5개 생성
4. 유한한 양의 용암이 끝까지 흐름
5. 끝까지 도달한 뒤 냉각 시작
6. 모든 용암이 현무암으로 굳음
7. 다시 분출 버튼 활성화

두 번째 분출에서도 기존 현무암은 삭제하지 않는다. 새 용암은 새로운 경로로 흐르며 기존 현무암 위를 지나갈 수도 있다.

## 9. 현무암 누적과 성능

각 분출이 흐르는 동안에는 갈래별 오브젝트가 필요하다. 완전히 굳은 뒤에는 모든 갈래와 기존 현무암을 하나의 지오메트리로 병합한다.

지오메트리 병합 효과:

- 삼각형 수와 정점 메모리는 크게 줄지 않는다.
- 오래된 현무암의 드로우콜을 1회로 유지할 수 있다.
- JavaScript 오브젝트와 머티리얼 관리 비용을 줄인다.

누적 현무암이 안전 삼각형 기준에 도달하면 추가 분출을 제한해 태블릿이 갑자기 멈추지 않도록 한다.

## 10. 화산가스와 폭발 효과

- 분화구 연기: 64개 Points 입자
- 용암 가스: 36개 Points 입자
- 용암 표면 기포: 가스 발생 위치를 공유하는 밝은 Points
- 폭발: 분화구 위 플래시와 방사형 입자

기포와 가스는 냉각이 진행될수록 줄어들고 완전히 굳으면 사라진다.

## 11. 태블릿 성능 원칙

- PC에서도 태블릿과 같은 품질 설정 사용
- 픽셀 비율 최대 1.25
- 1K 텍스처만 사용
- 실시간 유체 시뮬레이션 사용 안 함
- 경로 계산은 분출 시작 순간에만 실행
- 움직이는 입자 약 100개
- 현무암 병합 후 드로우콜 약 5회 수준

## 12. 현재 구현 상태

완료:

- 절차적 화산과 PBR 지형
- 단발 폭발과 랜덤 용암 경로
- 부드러운 용암 전면
- Lava004 → Lava002 → Rock041 자동 전환
- 아래쪽 우선 냉각
- 용암 도달 완료 후 냉각 시작
- 화산가스, 기포, 일부 기공
- 반복 분출과 기존 현무암 유지
- 누적 현무암 지오메트리 병합
- 태블릿 공통 성능 설정
- 지하 단면과 화강암 결정 성장 장면
- 지표와 지하를 연결하는 카메라 이동
- 암석층, 불규칙한 마그마 저장 영역, 화도와 암맥

- 실제 태블릿 기기 성능 측정
